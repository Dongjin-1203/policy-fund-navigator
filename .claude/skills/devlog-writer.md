# devlog-writer 스킬

## 목적
개발 세션 마무리 시 오늘 작업 내용을 기반으로
일관된 양식의 개발 회고 문서를 자동으로 작성한다.
모든 프로젝트에서 범용적으로 적용 가능하도록
설계한다.

## 트리거 조건
아래 상황에서 반드시 이 스킬을 적용할 것:
- "회고 작성", "개발일지 작성", "devlog 써줘" 언급 시
- "오늘 작업 정리해줘", "오늘 작업 회고해줘" 언급 시
- 세션 마무리 시 "마무리하자", "오늘 마무리" 언급 시
- "지난 회고와 같은 양식으로 써줘" 언급 시
- "개발 로그 남겨줘", "작업 기록 남겨줘" 언급 시

## 실행 순서

### Phase 1. 기존 회고 양식 파악

1. docs/devlog/ 또는 docs/ 또는 루트에서
   기존 회고 파일(.md) 존재 여부 확인.

   [기존 회고 있는 경우]
   - 가장 최근 파일 1개를 읽고 양식 파악
     (섹션 구조, 말투, 표 형식, 태그 등)
   - 시리즈 넘버링 방식 파악
     (#1, #2, #2-1, #2-2 등)
   - 다음 넘버링 자동 결정

   [기존 회고 없는 경우]
   - 아래 기본 양식 사용
   - 넘버링 #1부터 시작

### Phase 2. 오늘 작업 내용 파악

2. git log 기반 작업 내용 수집

   ① git log --oneline --since="yesterday"
      오늘 날짜 기준 커밋 목록 확인

   ② git log --oneline -20
      최근 커밋 이력으로 오늘 작업 범위 추정

   ③ git diff HEAD~{오늘커밋수}..HEAD --stat
      변경된 파일 목록 확인

3. 대화 맥락 기반 작업 내용 수집
   현재 대화에서 언급된 아래 항목 정리:
   - 오늘의 목표
   - 완료된 작업
   - 발견된 문제 및 해결 방법
   - 미완성 작업 및 다음 스텝

### Phase 3. 회고 문서 작성

4. 기본 양식으로 작성할 것.
   기존 회고가 있으면 해당 양식을 우선 따를 것.

---
기본 양식:

# [시리즈명 #N] {프로젝트명} — {오늘 작업 핵심 한 줄}

> 시리즈: {시리즈명}
> 태그: `태그1` `태그2` `태그3`
> 날짜: YYYY-MM-DD

---

## 1. 오늘의 목표

| 목표 | 결과 |
|---|---|
| 목표1 | ✅ / ❌ / ⚠️ |

---

## 2. 주요 작업 내용

### 2-1. {작업명}
내용

### 2-2. {작업명}
내용

---

## 3. 발견된 문제 및 해결 방법

### 🔴 Issue 1 — {문제명}
**문제**: ...
**해결**: ...

### 🟡 Issue 2 — {문제명} (미해결)
**문제**: ...
**현재 방침**: ...

---

## 4. 남은 작업

### 단기 (다음 세션)
- [ ] 작업1

### 중기
- [ ] 작업1

### 미결
- [ ] 작업1

---

## 5. 배운 점 / 느낀 점

(자유 서술)

---

*전체 코드: {저장소 URL}*
---

5. 작성 시 주의사항
   - 말투와 문체는 기존 회고를 따를 것
   - 해결된 문제는 🔴, 미해결은 🟡로 구분
   - 목표 결과는 ✅(완료) / ❌(미완) / ⚠️(부분완료)
   - 배운 점은 단순 나열이 아닌
     이유와 맥락이 담긴 서술로 작성

### Phase 4. 파일 저장

6. 파일명 결정
   형식: YYYY-MM-DD-{작업핵심키워드}.md
   오늘 날짜 자동 적용

7. 저장 경로 결정
   - 기존 회고 파일이 있는 경로와 동일하게 저장
   - 없으면 docs/devlog/ 에 저장
   - 디렉토리 없으면 생성

8. 저장 완료 후 문서 전문 출력

### Phase 5. 커밋 여부 확인

9. 저장 완료 후 사용자에게 확인
   "회고 파일을 커밋하시겠습니까?
   (docs/devlog/ 경로는 통상 .gitignore 처리)"

   [커밋 원하는 경우]
   git add {파일경로}
   git commit -m "docs: add devlog YYYY-MM-DD"

   [커밋 원하지 않는 경우]
   "저장 완료. 커밋은 생략합니다."로 표기

## 주의사항
- 기존 회고 양식이 있으면 반드시 그 양식을 따를 것
- 시리즈 넘버링은 기존 마지막 번호에서 자동으로 이어갈 것
- 작성 전 오늘 작업 내용을 사용자에게 확인받지 않고
  git log와 대화 맥락 기반으로 자동 파악할 것
- 배운 점 섹션은 생략하지 말 것
